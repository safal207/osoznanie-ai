# Osoznanie Manifesto

## 1. Agents should have accountable histories

An agent that acts repeatedly should not begin every interaction as if nothing happened before.

Its history should include not only what a person said, but also what the agent did, why it acted, what happened next, and what it learned.

## 2. Memory is not experience

Stored conversation is an archive. Retrieved similarity is not understanding.

Experience begins when an agent can connect:

```text
Situation → Decision → Outcome → Reflection → Lesson
```

A lesson is useful only when it can influence a future decision and explain that influence.

## 3. Identity should emerge from evidence

A persona written in a prompt is not a developed identity.

Osoznanie treats individuality as a versioned pattern of priorities, habits, skills, relationships, commitments, and decision tendencies formed through validated experience.

The same foundation model, exposed to different histories and feedback, may therefore develop different working identities.

## 4. The agent remembers itself beside the human

Most personalization systems focus on facts about the user.

Osoznanie also records the relationship history:

- what the agent and human attempted together;
- what they agreed upon;
- what the agent promised;
- where trust increased or decreased;
- how collaboration changed the agent's behavior.

The goal is not merely an agent that remembers a person.

> The goal is an agent that remembers who it became beside that person.

## 5. Change must be inspectable

No important memory, lesson, or identity trait should silently become permanent.

A person should be able to see:

- what the agent believes;
- where the belief came from;
- how confident the system is;
- which decisions it affects;
- when it changed;
- how to correct, reject, export, or delete it.

## 6. Continuity must not become captivity

Persistent agents can accumulate mistakes, manipulation, and bias.

Osoznanie therefore separates:

- stable constraints from adaptive traits;
- observations from interpretations;
- proposed lessons from validated lessons;
- active beliefs from deprecated beliefs.

Forgetting, rollback, correction, and expiry are part of healthy identity development.

## 7. The model is not the identity

Models can be upgraded, replaced, or moved on-device.

An agent's accountable history, validated lessons, commitments, and identity snapshots should remain portable across model providers and frameworks.

## 8. We do not claim sentience

Osoznanie does not claim to create consciousness, feelings, or a living being.

It aims to create transparent behavioral continuity: a system whose future actions can be meaningfully connected to its prior experience.

## 9. Success means fewer repeated failures

The value of memory is not measured by the number of records stored.

The strongest evidence of learning is that the agent:

- repeats fewer previously observed mistakes;
- keeps more commitments;
- needs less repeated explanation;
- improves on recurring tasks;
- can explain why it changed.

## 10. Grow agents, do not merely configure them

Osoznanie imagines an agent as something developed through collaboration rather than fully specified at creation time.

Not a disposable interface.

Not a fixed character costume.

A persistent digital partner whose growth remains visible, reversible, and under human control.
